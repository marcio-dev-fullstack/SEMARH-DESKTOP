======================================================================
 GUIA RÁPIDO DE EXECUÇÃO E TESTE - SEMARH FISCALIZA
======================================================================

Siga estes passos no terminal, dentro da pasta do projeto, para configurar e rodar a aplicação.

---
### PRÉ-REQUISITO: VERIFICAR A ESTRUTURA DO PROJETO
---

# Antes de começar, certifique-se de que você está na pasta raiz do projeto completo.
# Se você clonou do GitHub, verifique se está na branch correta (ex: 'main' ou 'master').
#
# A pasta raiz DEVE conter os seguintes diretórios e arquivos:
#   - Pastas: app, bootstrap, config, public, resources, routes, etc.
#   - Arquivos: artisan, composer.json
#
# Se essas pastas e arquivos não estiverem presentes, você provavelmente baixou um pacote
# incompleto ou está na branch errada.

---
### PASSO 1: CONFIGURAR O AMBIENTE
---

# 1.1. Copie o arquivo de exemplo de ambiente. O comando varia dependendo do seu sistema operacional.

# Para Windows (CMD ou PowerShell):
# Se estiver usando o PowerShell (padrão no VS Code):
Copy-Item .env.example .env

# Se estiver usando o CMD (Prompt de Comando) clássico:
copy .env.example .env

# Para Linux ou macOS:
cp .env.example .env

# 1.2. Abra o arquivo ".env" que você acabou de criar e configure as variáveis do banco de dados (DB_*).
#      Exemplo para PostgreSQL:
#      ------------------------------------
#      DB_CONNECTION=pgsql
#      DB_HOST=127.0.0.1
#      DB_PORT=5432
#      DB_DATABASE=semarh_fiscaliza
#      DB_USERNAME=postgres
#      DB_PASSWORD=sua_senha_secreta_aqui
#      ------------------------------------


---
### PASSO 2: INSTALAR DEPENDÊNCIAS E PREPARAR A APLICAÇÃO
---

# NOTA IMPORTANTE PARA USUÁRIOS WINDOWS:
# Os comandos a seguir usam 'php'. Se você receber um erro como "'php' não é reconhecido...",
# significa que o PHP não está no PATH do seu sistema.
#
# SOLUÇÃO: Adicione a pasta onde você instalou o PHP (ex: C:\php) às suas
# "Variáveis de Ambiente" do Windows e reinicie o terminal.
#

# 2.1. Instale as dependências do PHP (Laravel).
composer install

# 2.2. Gere a chave de criptografia da aplicação.
php artisan key:generate

# 2.3. Crie as tabelas no banco de dados.
php artisan migrate

# 2.4. Popule o banco de dados com dados de exemplo (usuários, processos, etc.).
php artisan db:seed

# 2.5. Crie o link simbólico para o armazenamento de arquivos (uploads).
php artisan storage:link


---
### PASSO 3: EXECUTAR O SERVIDOR DE DESENVOLVIMENTO
---

# 3.1. Inicie o servidor local. Por padrão, a aplicação estará acessível em http://127.0.0.1:8000
php artisan serve


---
### PASSO 4: TESTANDO COMANDOS AGENDADOS (Opcional)
---

# Para testar o envio de notificações de licenças a vencer sem esperar o agendador, execute o comando manualmente:
php artisan warnings:send-license-expiration
